librdf Frequently Asked Questions (FAQs)

  • What is the license?

    librdf is free software, or Open Source if you prefer.

  • Why is the code released under two licenses - LGPL and BSD?

    I originally considered just a GPL version of the library but I considered this too restrictive, hence the LGPL version. LGPL also allows the user, at their choice, to license the software under the GPL (See LGPL Term 3) in which case this software can also be used with all the other GPLed code -- see next answer.

    As an alternative, the entire library can be used under the terms of the modified BSD style license which allow you use it in whatever way you like as long as you don't pretend you wrote it, or use the authors name to promote it.

  • Can I link this library with code under license XXX?

    Firstly, IANAL however I here is an overview of how you can use this software:

    • If you choose the LGPL/GPL licenses, the GNU project list of licenses page says that you can link the following licensed software with the code:
      GPLed, LGPLed, X11, original BSD, modified BSD, Perl (as GPL), zlib
      and not these types:
      Apache, Zope, IBM, LaTeX (LPPL), MPL, NOSL, Sun PL, NPL, QPL, Artistic, APPL, Sun CSL, Plan 9
      check the link above for updates, this is probably not a definite list.
    • If you choose the modified BSD license, you should be able to link with most anything else except you might have problems distributing stuff which includes GPLed/LGPLed parts, in which case, choose the LGPL or GPL licenses.
  • Why is librdf written in C?

    Writing in C to allow reuse in many ways and can be used in many languages since most of them allow importing of C libraries or calling of C functions.
